Backdesk is a workspace for structured data. Users create dashboard pages, collection pages, and imported system collections. The current product is generic at the platform layer, with Interactive Brokers activity statement import as the first vertical workflow.

These docs describe the system implemented in this repository today. The older v1 product direction remains in `docs/superpowers/specs/2026-04-28-backdesk-v1-design.md`, but these Mintlify pages should be treated as the current engineering reference.

## Current stack

- Next.js 15 App Router in `apps/web`
- React 19, shadcn/ui, Tailwind CSS v4, and Plate.js for rich dashboard editing
- Supabase Auth, Postgres, Storage, and Row-Level Security
- Turborepo and pnpm workspaces
- Vitest for unit tests, Playwright for browser tests, Lighthouse CI for quality gates
- Sentry and Vercel Analytics instrumentation

## Main domains

- **Pages:** Sidebar entities with either `dashboard` or `collection` page type.
- **Dashboards:** Plate documents stored as JSON on `pages.document`.
- **Collections:** Typed field definitions plus JSONB row data.
- **Blocks:** Generic dashboard blocks that read from collections.
- **Connections:** Importers that create or update system collections.
- **Settings:** Account, profile, appearance, and connections surfaces.
